### 경쟁사 도출 방식 (3단계 진화)

| 단계 | 방식 | 결과 |
|------|------|------|
| 1차 | 카테고리 co-occurrence 자동 계산 | 실패 — 바이너리샵(407회) 등 특정 브랜드 과대 집계 |
| 2차 | 카테고리 화이트리스트 필터 적용 | 부분 개선, 렌탈 카테고리 혼재 문제 잔존 |

**COMPETITOR_MAP 구성:**
- 1차 72개 브랜드 수동 작성 (주요 가구·가전·식품·생활용품 등)
- 2차 363개 브랜드 AI 검토 후 카테고리 기반 일괄 지정
- 최종 총 435개 브랜드 경쟁사 3개씩 지정 / 7개 공란 (브랜드명 아닌 항목)

- 네이버 브랜드스토어 / 스마트스토어 제외
- 올리브영, 오늘의집 등 타 플랫폼 내 브랜드 페이지 제외

## 6. 개발 이슈 및 해결

| 이슈 | 원인 | 해결 방법 |
|------|------|-----------|
| networkidle 타임아웃 | 오늘의집 SPA 특성상 네트워크가 완전히 종료되지 않음 | `waitUntil: 'domcontentloaded'`로 변경 |
| 브랜드명 대신 속성값 수집 | 브랜드 필터 드롭다운에 사이즈·색상 등 혼재 | 다층 노이즈 필터 구현 (NOISE_EXACT / PATTERN / SPACE_ATTR / GENERIC) |
| 경쟁사 특정 브랜드 과대 집계 | 바이너리샵(407회), 살림의달인(383회) 등이 거의 모든 카테고리에 등장 | 카테고리 자동 계산 폐기 → 하드코딩 맵으로 전환 |
| Excel 경로 ENOENT | 스크립트 실행 시 shell CWD가 리셋됨 | `path.join(__dirname, ...)` 절대경로로 수정 |
| apply_competitors.js 컬럼 인덱스 오류 | 기존 Excel에 홈페이지 컬럼(D열) 존재 미확인 | 헤더에서 `indexOf('홈페이지')`로 동적 인덱스 탐지 방식으로 수정 |
