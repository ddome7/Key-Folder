# 피드백 시스템 추가 개발 내역
> 작업일: 2026-04-10

## 작업 개요
밀리 파트너스 프로젝트의 피드백 기능을 평가 시스템(index.html)에 이식.
관리자 전용으로 제한하고, 평가 시스템 내부에 피드백 관리 페이지 추가.

---

## 1. feedback_test.html 테스트 파일 생성
- 경로: `C:\Users\퍼즐\Desktop\LJM_Test\feedback_test.html`
- 평가 시스템 UI 스타일(파란색 테마 #1C3FAA)로 피드백 CSS 변환
- Supabase page_id = 'performance'으로 분리 테스트
- 테스트 후 정상 동작 확인 → 실제 프로젝트에 반영

## 2. index.html 피드백 시스템 삽입 (관리자 전용)
**변경 파일:** `C:\Users\퍼즐\Desktop\LJM_Test\index.html`

### 추가된 기능
- 우측 하단 ✏️ 피드백 토글 버튼
- 피드백 모드 ON/OFF (섹션 클릭 → 우측 패널 슬라이드)
- 섹션 피드백 / 요소 피드백 2가지 모드
- 이미지 첨부 (Supabase Storage `feedback-images` 버킷)
- Supabase `page_feedbacks` 테이블, `page_id = 'eval-system'`

### 관리자 게이트 방식 변경 이력
1. localStorage 방식 (`localStorage.setItem('fb_admin','true')`) → 초기 구현
2. **isAdminRole() 자동 연동** → 최종 채택
   - 로그인 후 `appUser.role`이 admin/team1_lead/team2_lead/head_lead이면 자동 노출
   - member/viewer 계정에는 버튼 완전 미노출

### SECTION_MAP (page_id 기반)
```javascript
'page-dashboard'   → 대시보드
'page-team-sales'  → 팀 매출 현황
'page-members'     → 팀원 평가
'page-eval'        → 팀장 평가
'page-advertisers' → 광고주 관리
'page-criteria'    → 평가 기준
'page-settings'    → 시스템 설정
'page-report'      → 리포트
```

## 3. 피드백 관리 페이지 추가 (사이드바)
- 사이드바에 📬 피드백 관리 메뉴 추가 (`nav-admin-only` 클래스)
- `page-feedback-mgmt` 페이지 HTML 추가
- `PAGE_NAV_MAP`에 `'feedback-mgmt': 'nav-feedback-mgmt'` 추가
- `showPage()` 함수에 `feedback-mgmt` 케이스 추가

### 피드백 관리 기능
- 섹션 / 우선순위 / 상태 필터
- 상태 변경 인라인 처리 (대기 → 검토 중 → 완료)
- 피드백 삭제
- 첨부 이미지 클릭 → 원본 확인

## 4. Supabase 계정 삭제 (테스트 계정 정리)
- 기존 테스트 이메일 계정이 중복 가입 불가 상태
- Supabase 대시보드 → Authentication → Users에서 직접 삭제 안내
- 삭제 후 재가입 가능 확인
