# 역량강화 항목별 승인 요청/승인하기 기능 개발내역

> 작성일: 2026-04-03
> 커밋: `de4d3eb`

---

## 작업 배경

기존에는 역량강화 항목 전체를 한 번에 "팀장 승인 요청"하는 섹션 단위 버튼이 있었음.
사용자 요청에 따라 **항목 하나하나를 개별 승인 요청**할 수 있도록 전면 개선.

---

## 변경 내역

### 1. 항목별 승인 요청 버튼 (팀원 화면)

- 각 역량강화 항목의 `1점 / 2점 / URL / 메모` 버튼 **맨 오른쪽 끝**에 버튼 추가
- 미요청 상태: **승인 요청** (파란색)
- 요청 완료 상태: **✅ 요청됨** (초록색) — 다시 클릭하면 요청 취소(토글)
- 요청 상태는 Supabase `skill_approvals.requested_item_ids` 에 항목 ID 목록으로 저장

### 2. 항목별 승인하기 버튼 (팀장 화면)

- 팀원이 승인 요청한 항목에만 **승인하기** (노란색) 버튼 표시
- 클릭 시 승인자 역할에 따라 체크박스 자동 체크:

  | 승인자 역할 | 자동 체크 |
  |-------------|-----------|
  | team1_lead | 1팀장 체크박스 ✓ |
  | team2_lead | 2팀장 체크박스 ✓ |
  | admin / head_lead | 양쪽 모두 ✓ |

- 양쪽 팀장 모두 승인 완료 시 항목 **상단 고정 + 잠금** (기존 동작 유지)

### 3. Supabase 연동 로직

- `requestItemApproval(itemId)`: DELETE → INSERT 패턴으로 upsert
  - `team`, `requested_item_ids` 포함해서 저장
  - 요청 항목이 0개가 되면 행 전체 삭제
- `approveSkillItem(itemId)`: PATCH로 `requested_item_ids`에서 해당 항목 제거
  - 남은 대기 항목 없으면 행 삭제

### 4. 대시보드 승인 대기 카드

- 기존: 이름 + 요청 일시 + "평가 입력으로 이동" 버튼
- 변경: **N개 대기** 뱃지 추가 (노란색 배경)

### 5. 제거된 항목

- 섹션 단위 "팀장 승인 요청" 버튼 (member 전용 div)
- 섹션 단위 "확인 완료" 버튼 (admin 전용 div)
- `requestSkillApproval()` 함수
- `clearSkillApproval()` 함수

## 수정된 함수 목록

| 함수 | 변경 유형 |
|------|-----------|
| `makeItem()` | `approvalRequested: false` 필드 추가 |

| `updateSkillApprovalUI()` | 섹션 단위 show/hide 제거, `approvalRequested` 플래그 세팅으로 단순화 |
| `renderSkillApprovalDashboard()` | `requested_item_ids` 조회 추가, 대기 건수 뱃지 렌더 |
| `requestItemApproval()` | 신규 추가 |
| `approveSkillItem()` | 신규 추가 |
| `requestSkillApproval()` | 삭제 |
| `clearSkillApproval()` | 삭제 |
