## 3. 평가 구조

### 3개 영역 총 100점 만점

| 영역 | 배점 | 평가 방식 |
|------|------|-----------|
| ① 매출 강화 | 60점 | 월별 취급고 수동 입력 → 등급 자동 환산 |
| ② 성과 강화 | 25점 | 6개 항목 평가 (KPI 달성률 포함 일부 자동 계산) |
| ③ 역량 강화 | 15점 | 가점/감점 항목 직접 선택 |

### 최종 등급 기준
**S** 90점 이상 / **A** 80~89점 / **B** 70~79점 / **C** 60~69점 / **D** 60점 미만

### 반기 구분
- **H1 상반기**: Q1(12·1·2월) + Q2(3·4·5월)
- **H2 하반기**: Q3(6·7·8월) + Q4(9·10·11월)

## 5. 주요 기능 목록

### 대시보드
- 전체 팀원 평가 현황 카드 (점수·등급·진행률 바)
- **H1 / H2 반기 탭** — 반기별 제출·임시저장 데이터 분리 표시
- 팀 평균 카드 — 등급(대·컬러) + 점수(서브) 병렬 표시
- 역량강화 승인 대기 카드 (팀원별 N개 대기 뱃지)
- 평가 대상 아님(수습·인턴) 인원 붉은색 표시

### 팀원 · 광고주 관리
- 팀원 추가·수정·삭제 (재직/퇴사/임시 상태, 직책 포함)
- 드래그앤드롭으로 팀원 순서 자유 변경 (같은 팀·상태 그룹 내)
- 광고주 추가·수정 (운영중/종료 구분, 팀별 필터 탭)
- 월별 취급고 입력 (2023~2026)
- 광고주 이관(담당자 변경) 이력 관리
- 설정 가입인원 목록 순서 드래그앤드롭 변경

### 팀 취급고 현황
- **전체보기** 탭 (기본값) — 월별 취급고 + 수수료 수익 + 재직자 한 화면
- **월별 취급고** 탭
- **수수료 수익** 탭 — 1팀/2팀 월별 수기 입력, 합계·평균 인당 수익 자동 계산
  - 1팀원(수) / 2팀원(수) 월별 팀원 수 입력 (입사·퇴사 반영)
  - 🔍 셀별 메모 버튼
  - 1팀·2팀 증빙 Google Sheets 링크 버튼
- **재직자** 탭

### 매출 강화 평가 (60점)
- 12개월 취급고 수동 입력
- 당분기 / 전분기 취급고 자동 합산
- 취급고 상세 팝업 (돋보기 버튼) — 당분기·전분기 모두 지원

### 성과 강화 평가 (25점)
- **KPI 달성률** (광고주 카드 단위)
  - 광고주별 KPI 항목 최대 5개 (핵심·보조 구분)
  - 핵심 KPI 가중 70%, 보조 KPI 가중 30%
  - 전략 중요도 가중 (S=1.5배 / A=1.0배 / B=0.7배)
  - KPI 방향성: higher(실적/목표) / lower(목표/실적), 150% 상한
  - 도전 목표 보너스 자동 계산
  - 월별 탭 UI (분기 내 각 월 독립 입력)
  - 700만원 미만 광고주 자동 제외
  - 광고주별 KPI 평가 제외 기능 (빨간 제외 버튼·태그·복원)
  - KPI 카드 0개 시 분모 자동 조정 (30→25점)
- **예산 증액률** 자동 계산
- 기타 항목 수동 선택 (콘텐츠기획·보고서품질·무사고운영·협업)
- 가점 2개 항목 (+2pt × 2)

### 역량 강화 평가 (15점)
- 기본 3점 + 가점/감점 항목 선택
- 증빙 URL / 메모 첨부 (다중 URL 지원)
- **항목별 팀장 승인 요청·승인하기** (팀원/팀장 역할 분리)
  - 팀원: 항목별 "승인 요청" 토글 버튼
  - 팀장: 요청된 항목에 "승인하기" → 역할별 체크박스 자동 체크
  - 양쪽 팀장 승인 완료 시 항목 상단 고정 + 잠금

### 팀리더 평가 페이지
- 일반 매니저 평가 레이아웃 대신 별도 페이지 표시
- 수수료 수익 테이블 (읽기 전용)
- 해당 팀 소속 매니저 평가 현황 테이블 (현재 반기 기준)
- 팀 요약 카드 (전체 팀원 수 / 평가 완료 / 임시저장 / 팀 평균)
- 평가 대상 아님 인원 붉은색 표시

### 계정·권한 관리
- Supabase Auth 기반 이메일/비밀번호 로그인
- 신규 가입 → 관리자 승인 후 로그인 허용 (approval_status)
- viewer 역할 — 전체 열람 가능, 수정·삭제 불가 (body.viewer-mode CSS 제어)

### 데이터 관리
- Supabase 자동 저장·로드 (다기기 동기화)
- localStorage 폴백
- CSV 백업 (팀원·광고주 개별)
- JSON 전체 백업·복구

## 6. 데이터 구조

```
localStorage / Supabase (app_state)
├── appState
│   ├── TEAM_MEMBERS[]          — 팀원 목록 (이름·직급·직책·상태·팀)
│   ├── ADVERTISERS[]           — 광고주 목록 (이름·팀·상태·이관이력)
│   ├── budgetData{}            — 광고주별 월별 취급고 (2023~2026)
│   ├── evalData{}              — 팀원별 평가 데이터 (분기별)
│   ├── kpiState{}              — KPI 달성률 상태 (분기·담당자별)
│   │   └── [분기_담당자ID]
│   │       ├── [광고주ID][]    — KPI 항목 배열
│   │       └── __excluded[]   — 제외된 광고주 ID 목록
│   ├── skillItems{}            — 역량 강화 항목 (팀원별)
│   ├── submittedReports{}      — 최종 제출 리포트 (halfIdx 포함)
│   ├── feeRevenue{}            — 수수료 수익 (fyId·팀·월별)
│   │   ├── [fyId][team][month] — 팀별 월별 수익
│   │   ├── headcount1/2[month] — 1팀/2팀 월별 팀원 수
│   │   └── headcountMemo1/2[month] — 셀별 메모
│   └── profileDisplayOrder[]  — 설정 가입인원 표시 순서

Supabase 별도 테이블
├── profiles                    — 사용자 프로필 (role, team, approval_status)
└── skill_approvals             — 역량강화 항목별 승인 상태
    ├── member_name / team_member_id / team
    ├── half_key                — 평가 반기 식별자
    ├── skill_data (JSONB)
    └── requested_item_ids (JSONB)
```
