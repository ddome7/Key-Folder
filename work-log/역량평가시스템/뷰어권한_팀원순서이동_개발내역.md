## 기능 1 — 보기권한 (viewer) 역할 추가

### 개요
관리자와 동일하게 모든 페이지/내용을 볼 수 있되, 수정/삭제/제출 등 쓰기 동작은 불가한 `viewer` 역할 추가.

### 변경 내역

#### 역할 함수 업데이트
- `getRoleLabel()`: `viewer` → `'보기권한'` 매핑 추가
- `getRoleBadgeStyle()`: 뷰어 배지 스타일 추가 (하늘색 계열)

#### `applyRoleUI()` 수정
- `isViewer` 감지 후 `document.body.classList.toggle('viewer-mode', isViewer)` 적용
- `nav-admin-only` 네비 메뉴: admin **또는 viewer**이면 표시
- `header-admin-btns` (데이터 가져오기/내보내기 등): viewer에게는 숨김

#### CSS — `body.viewer-mode` 규칙 추가 (line ~1320)
```css
body.viewer-mode .member-edit-btn,
body.viewer-mode .member-del-btn,
body.viewer-mode .member-move-btn,
body.viewer-mode .adv-del-btn,
body.viewer-mode .skill-add-trigger,
body.viewer-mode .kpi-add-row-btn,
body.viewer-mode .kpi-card-delete-btn,
body.viewer-mode .kpi-row-del-btn,
body.viewer-mode #header-admin-btns,
body.viewer-mode .action-row { display: none !important; }
body.viewer-mode .eval-layout { pointer-events: none; opacity: 0.85; }
body.viewer-mode #sales-manager-select-header { pointer-events: auto !important; ... }
```
- 평가 입력 레이아웃 전체 `pointer-events: none` → 수정 불가
- 단, 담당자 선택 드롭다운은 재활성화(매니저 전환해서 조회는 가능)

#### 설정 페이지 드롭다운
- 권한 선택에 `<option value="viewer">보기권한</option>` 추가

#### 대시보드 승인 카드
- `renderSkillApprovalDashboard()`: viewer도 카드 목록 볼 수 있도록 조건 완화
  - (승인 버튼은 role 조건에 따라 렌더되지 않음 — 별도 처리 불필요)

## 기능 2 — 팀원 순서 이동

### 개요
팀원 관리 목록에서 ▲/▼ 버튼으로 같은 팀·같은 상태(재직/퇴사) 그룹 내 순서를 자유롭게 변경할 수 있음. 순서는 앱 상태에 저장되어 영구 반영.

### 변경 내역

#### `memberRowHtml()` 수정
버튼 영역 끝에 이동 버튼 추가:
```html
<button class="member-move-btn" onclick="moveMemberInList(${m.id}, -1)" title="위로">▲</button>
<button class="member-move-btn" onclick="moveMemberInList(${m.id},  1)" title="아래로">▼</button>
```
- `.member-move-btn` CSS: 회색 미니 버튼 스타일
- viewer-mode에서 자동 숨김 (CSS 규칙)

#### `moveMemberInList(memberId, dir)` 함수 추가 (line ~3853)
1. 해당 멤버의 팀+상태가 같은 그룹만 추출
2. 그룹 내 인덱스 ± dir 으로 swap 대상 결정
3. `TEAM_MEMBERS` 배열에서 실제 인덱스 swap
4. `renderMemberList()` 재렌더 + `saveAppState()` 저장
