# 로그인 로딩 무한 대기 수정 개발내역

> 작성일: 2026-04-07
> 커밋: `066c1b4`, `9e7a6b8`

---

## 작업 배경

F5 새로고침 시 로딩 화면에서 넘어가지 않는 현상이 자주 발생.
`fetchUserProfile` 함수가 Supabase 호출 중 응답이 느릴 경우 무한 대기 상태에 빠지는 것이 원인.

---

## 수정 1 — fetchUserProfile 타임아웃 + 재시도 추가

### 원인

`fetchUserProfile(userId)`가 단순 `await` 호출이라 네트워크 지연 시 무한 대기.

### 수정 내용

`Promise.race`로 6초 타임아웃 적용, 실패 시 최대 3회 재시도:

```js
async function fetchUserProfile(userId, retries = 3) {
  for (let attempt = 0; attempt < retries; attempt++) {
    try {
      const result = await Promise.race([
        sbClient.from('profiles').select('name, role, team, approval_status').eq('id', userId).single(),
        new Promise((_, reject) => setTimeout(() => reject(new Error('timeout')), 6000))
      ]);
      const { data, error } = result;
      if (error) { console.warn('[fetchUserProfile] error:', error.code, error.message); return null; }
      return data || null;
    } catch(e) {
      console.warn('[fetchUserProfile] attempt', attempt + 1, '실패:', e.message);
      if (attempt < retries - 1) await new Promise(r => setTimeout(r, 800 * (attempt + 1)));
    }
  }
  console.error('[fetchUserProfile] 3회 재시도 모두 실패');
  return null;
}
```

- 1회: 6초 타임아웃 → 실패 시 800ms 대기
- 2회: 6초 타임아웃 → 실패 시 1600ms 대기
- 3회: 6초 타임아웃 → 실패 시 null 반환 → 로그인 화면 이동

## 수정 2 — 전역 로딩 가드 추가 후 제거

### 추가 시도

`DOMContentLoaded`에서 12초 타임아웃 + `window.showAppScreen` 래퍼 방식으로
로딩 안전망 구현 시도.

### 부작용

`window.showAppScreen` 재정의가 새로고침 시 페이지 복원 로직을 방해,
대시보드로 강제 이동되는 현상 발생.

### 결론

전역 가드 + 래퍼 전체 제거. `fetchUserProfile` 재시도/타임아웃만으로 충분히 해결됨.
