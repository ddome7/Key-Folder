# 평가기간 전환 버그 수정 개발내역

> 작성일: 2026-04-06
> 커밋: `0ddbd44`, `f6ec0e6`

---

## 작업 배경

평가 입력 탭에서 평가기간(H1/H2, 회계연도)을 변경했을 때
매출값과 KPI 달성율 등이 새 기간에 맞게 갱신되지 않는 버그 수정.

---

## 수정 1 — 분기 전환 시 이전 분기 데이터 오염 방지

### 증상
H1 Q1에서 H2로 전환하면 성과강화 셀렉트가 H1 값을 그대로 보여줌.
KPI 달성율도 이전 분기 기준으로 계산됨.

### 원인
`_restoreAfterQuarterSwitch()` 내부 실행 순서 문제:

```
(기존)
1. 매출 입력 채우기
2. calcSalesAvg()  ← 내부에서 updateTotal → savePerformState 호출
   → 이 시점에 DOM은 아직 이전 분기 값 → 새 분기 키에 오염 저장
3. 성과강화 DOM 초기화  (이미 늦음)
4. restorePerformState()  ← 오염된 상태 복원
```

### 수정 (`_restoreAfterQuarterSwitch`)

```
(수정 후)
1. 성과강화 DOM 먼저 초기화 (select 리셋, 보너스 언체크)
2. 매출 입력 채우기
3. restorePerformState()  ← 새 분기 올바른 상태 복원
4. calcSalesAvg()         ← 올바른 DOM 상태로 점수 계산
   (calcBudgetGrowth·renderSalesChart 중복 호출 제거 — calcSalesAvg 내부에서 처리)
```

---

## 수정 2 — 회계연도 변경 시 이전 데이터 잔류 수정

### 증상
회계연도를 변경해도:
- 매출 평균·등급·점수가 이전 연도 값 그대로 표시
- 오른쪽 매출 차트가 이전 연도 데이터 유지
- KPI 카드가 이전 연도 카드 그대로 남아있음
- 타 매니저를 클릭해야만 새 연도 데이터가 반영됨

### 원인
`onFiscalYearChange()`에서 우측 패널은 초기화하지만
매출 강화 섹션(평균·등급·점수), 차트, KPI 카드 컨테이너를
명시적으로 초기화하지 않음.

### 수정 1 — 매출·차트·KPI 카드 초기화 추가

`onFiscalYearChange()` 내 역량강화 초기화 직전에 추가:

```js
// 매출 강화 섹션 초기화 (평균·등급·차트)
salesAvgEl.textContent = '—';
salesGradeEl.textContent = '—';
salesScoreEl.textContent = '— / 60점';
salesFilledEl.textContent = '0';
renderSalesChart();

// KPI 카드 초기화
kpiContainer.querySelectorAll('.adv-card').forEach(el => el.remove());
kpiEmptyEl.style.display = '';
```

### 수정 2 — 선택된 매니저 자동 재로드

연도 변경 전 `currentEvalMemberId` 저장 후,
모든 초기화 완료 후 해당 매니저를 새 연도 기준으로 자동 재로드:

```js
const prevMemberId = currentEvalMemberId; // 변경 전 저장

// ... 기존 초기화 로직 ...

// 마지막에 자동 재로드
if (prevMemberId && TEAM_MEMBERS.find(m => m.id === prevMemberId)) {
  selH.value = prevMemberId;
  onManagerChange(prevMemberId);
}
```

## 수정된 함수

| 함수 | 변경 유형 |
|------|-----------|
| `_restoreAfterQuarterSwitch()` | DOM 초기화 순서 앞으로 이동, calcSalesAvg 위치 변경, 중복 calcBudgetGrowth/calcPerform/renderSalesChart 제거 |
| `onFiscalYearChange()` | 매출·차트·KPI 초기화 추가, prevMemberId 자동 재로드 로직 추가 |
